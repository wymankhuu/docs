## <Icon icon="universal-access" size="24" /> Our commitment

Playlab is built so that every learner and every educator can use it — regardless of how they interact with the web. We design and test against the **Web Content Accessibility Guidelines (WCAG) 2.0 at the AA conformance level**, the standard most widely referenced by schools, districts, and procurement teams.

Accessibility is treated as an ongoing investment, not a one-time audit. New features are reviewed against the same standards before they ship.

## <Icon icon="keyboard" size="24" /> Keyboard navigation

Every interactive element in Playlab can be reached and operated using only a keyboard.

- **Visible focus indicators** appear on every interactive element — buttons, links, inputs, menus — so you can always see where you are on the page.
- **Tab order** follows the visual reading order, so moving through a page with `Tab` and `Shift + Tab` matches what you'd expect.
- **Standard keys** (`Enter` to activate, `Esc` to dismiss, arrow keys inside menus) behave consistently across the product.

## <Icon icon="audio-description" size="24" /> Screen reader support

Playlab is designed to work with screen readers such as VoiceOver, NVDA, and JAWS.

- **Accessible labels on icon buttons** — every icon-only button (send, attach, flag, share, and so on) has a descriptive label so screen readers can announce what it does, not just "button".
- **Semantic structure** — pages use proper headings, landmarks, and lists so screen reader users can navigate by structure.
- **State changes are announced** — when something is selected, loaded, or expanded, screen readers are notified.

## <Icon icon="eye" size="24" /> Visual design

- **Color contrast** meets WCAG 2.0 AA minimums for body text and interactive elements.
- **Color is never the only signal** — status, errors, and required fields are communicated with text or icons as well as color.
- **Resizable text** — Playlab respects browser zoom and OS-level text-size preferences.

## <Icon icon="hand" size="24" /> Built-in accessibility features

Several Playlab features are themselves accessibility tools you can turn on as needed:

- [Voice Input](/features/Voice%20Input) — speak instead of typing, with real-time transcription
- [Voice Output](/features/Voice%20Output) — listen to Playlab's responses with text-to-speech
- [Using Dictation](/features/Using%20Dictation) — dictate longer responses with platform dictation
- [Spanish Mode](/features/Spanish%20Mode) — operate Playlab in Spanish
- [Math Inputs](/features/Math%20Inputs) — input math expressions without relying on keyboard symbols
- [Camera Handwriting Input](/features/Camera%20Handwriting%20Input) — submit handwritten work via camera

## <Icon icon="bullhorn" size="24" /> Reporting an accessibility issue

If something in Playlab is blocking you or a student, we want to hear about it. Accessibility issues are prioritized.

Contact us at [support@playlab.ai](mailto:support@playlab.ai) with:

- A short description of what you were trying to do
- The assistive technology you were using (if any)
- The browser and operating system
- A screenshot or screen recording if you can capture one
