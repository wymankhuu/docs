*How to monitor usage and engagement at the workspace level in V2.*

The Activity view lets workspace managers and org admins see how members are using the apps in a workspace. V2 keeps the V1 filtering and adds two things: per-class segmentation when an app is used in multiple workspaces, and a member detail page reachable from any activity row.

<Warning>
  **Access requirements**

  Activity views are visible to workspace managers and org admins. Members see their own activity from their dashboard.
</Warning>

## What activity shows

The Activity view inside a workspace lists every app session that took place there. For each session, you see:

- The app used
- The member who used it
- Start time and duration
- Message count
- Any flags raised during the session

You can click into any session to read the full transcript. Conversations are preserved in V2 the same way they were in V1.

## Filtering by app, user, date

The filter controls at the top of the view let you narrow by:

**App.** Show sessions for one specific app, or for a subset.

**Member.** Show sessions from one member.

**Date range.** Last 7 days, last 30 days, custom range.

**Flag status.** Show only flagged sessions.

Filters combine. Showing a single member's flagged sessions on one app is one path through the same view.

## Activity per class in V2

V2 segments activity by workspace. When the same app is used in three different classes (workspaces), each class sees only its own sessions. As an admin spanning multiple workspaces, you can switch the workspace selector to see each class separately, or use the org-level view for an aggregated picture.

This is the change that supports cross-workspace app use: a teacher running three sections of the same class adds one app to three workspaces and sees three segmented activity views.

## Drilling into a member

V2 adds a member detail page reachable from the Members tab or by clicking a member's name in any activity row. The detail page collects everything that member did inside the workspace: apps used, conversations, flags, and apps built.

For the full guide, see [Workspace member detail](https://learn.playlab.ai/features/Workspace%20Member%20Detail).

## Best practices

A few things teachers have asked about and that have worked:

Set a weekly review cadence. Open the Activity view every Friday and scan for sessions that stalled or members who have not engaged. Five minutes catches most issues.

Use date filtering to avoid being overwhelmed. The all-time view is rarely useful for monitoring. Last 7 days or last 14 days is the right window.

Pay attention to flags first. Filter by flag status when you open the view. Anything flagged is worth a quick read.

Open conversations that look unusual. A very long session for a brief assignment, or a very short session for a complex assignment, are signals worth investigating.

## FAQ

**Can I export activity data?**

Yes. The Export button on the Activity view downloads filtered data as CSV. Org admins can export from the org-level dashboard.

**Cross-class views?**

For viewing the same app across multiple classes you teach, switch the workspace selector or use the org-level view if you have admin access.

**What is the difference between this and org-level monitoring?**

Workspace activity is scoped to one class. Org-level monitoring (admins only) sees activity across every workspace in the org and surfaces flags from across the network. See [Centralized org monitoring](https://learn.playlab.ai/features/Reviewing%20App%20Activity).

**How long is activity history retained?**

For the lifetime of the workspace. Archived workspaces retain history. Deleted workspaces' activity remains visible to org admins for compliance and review.

**Can students see other members' activity?**

No. Members see their own conversations and their own apps. Cross-member visibility is a manager and admin role.
